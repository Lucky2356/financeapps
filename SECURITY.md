# Политика безопасности

## Поддерживаемые версии

Обновления безопасности выпускаются для последней версии. Используйте свежий релиз
с [страницы Releases](https://github.com/Lucky2356/financeapps/releases).

| Версия | Поддержка |
|---|---|
| Последний релиз | ✅ |
| Более старые | ❌ |

## Сообщить об уязвимости

**Пожалуйста, не сообщайте об уязвимостях через публичные GitHub Issues.**

Используйте приватный канал:

- **GitHub Security Advisories** — вкладка *Security → Report a vulnerability*
  ([создать приватный отчёт](https://github.com/Lucky2356/financeapps/security/advisories/new)).

В отчёте по возможности укажите:

- тип уязвимости и затронутый компонент (десктоп / веб / API-роут / зависимость);
- шаги воспроизведения или PoC;
- потенциальное влияние;
- предлагаемое исправление, если есть.

Мы постараемся подтвердить получение в течение нескольких дней и держать вас в курсе
хода исправления. Пожалуйста, дайте разумное время на выпуск патча до публичного
раскрытия (ответственное разглашение).

## Модель безопасности — кратко

- **Десктоп:** данные хранятся локально (IndexedDB) и не покидают устройство; сетевые
  обращения — только к публичным рыночным данным (MOEX ISS) и, при явном включении, к
  Anthropic (ИИ-ассистент).
- **Веб:** пароли хранятся только в виде bcrypt-хеша; данные изолированы по пользователю
  (фильтрация по `userId`); включён rate-limit; HTTPS терминируется через Caddy.
- **Секреты** (`AUTH_SECRET`, `POSTGRES_PASSWORD`, ключи подписи обновлений, API-ключи)
  **никогда** не коммитятся в репозиторий и задаются через переменные окружения.

## Что в зоне риска

- Утечка/изоляция пользовательских данных в веб-режиме
- Аутентификация и управление сессиями (NextAuth)
- Инъекции (SQL через Prisma, XSS), обход валидации (Zod)
- Цепочка обновления десктопа (подпись релизов)
- Уязвимости в зависимостях (`npm audit`)

Спасибо, что помогаете сохранять проект и его пользователей в безопасности. 🔐
